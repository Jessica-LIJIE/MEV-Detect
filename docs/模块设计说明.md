## 3. 核心数据结构（`src/types.py`）

### MarketSnapshot

监听器触发时传递给 PSO 的统一输入：

```python
@dataclass
class MarketSnapshot:
    snapshot_id: str
    timestamp: str
    trigger: str                # 如 "l1_large_swap"
    l1: PoolState
    l2: PoolState
    l2_lag_ms: int
    gas: GasState
    bridge_fee_usd: float
    swap_amount_usd: float | None = None
    scenario: str | None = None
```

### 粒子搜索空间

$$X = [Amount_{in},\ Route_{L1},\ Route_{L2},\ Bridge_{path}]$$

| 索引 | 参数 | 类型 | 范围 |
|------|------|------|------|
| 0 | `amount_in` | 连续 | [0.1, 100.0] ETH |
| 1 | `route_l1` | 离散编码 | [0, 2] |
| 2 | `route_l2` | 离散编码 | [0, 2] |
| 3 | `bridge_path` | 离散编码 | [0, 2] |

### SearchResult / ArbitrageStrategy

PSO 输出 `SearchResult`；经 `build_strategy()` 转为可展示的 `ArbitrageStrategy`（含 `summary()` 方法）。

### 4.1 listener.py

```python
class MultiChainListener:
    def __init__(self, use_http_poll: bool = False): ...

    async def start(self) -> None: ...
    async def stop(self) -> None: ...
    async def run_for(self, duration_sec: float) -> None: ...
    @staticmethod
    def is_configured() -> bool: ...
    @staticmethod
```

**职责：** 双链 WebSocket/HTTP 监听、Swap 解析、L1 大额触发、掉线重连、`l2_lag_ms` 计算。

### 4.2 optimizer.py

```python
class PSOOptimizer:
    def search(
        self,
        fitness_fn: Callable[[torch.Tensor], torch.Tensor],
        max_iter: int = 100,
        patience: int = 15,
    ) -> SearchResult: ...

def create_pso_optimizer(
    num_particles: int,
    bounds: torch.Tensor,
    device: str = "cpu",
    rank: int = 0,
    world_size: int = 1,
    **kwargs,
) -> PSOOptimizer: ...
```

**硬性约束：**

- 粒子更新、fitness 不得使用粒子级 `for`
- 多 GPU 时在 `search()` 内每轮同步 `gbest`【待实现】

## 5. 调用时序

### Mock 模式

```
main.py --mock
  → load_mock_snapshots()
  → for each snapshot:
       select_pso_profile(snapshot)  【待实现】
       PSOOptimizer.search(evaluate_fitness)
       build_strategy → log / 写 jsonl
```

### Live 模式

```
main.py --live
  → MultiChainListener.on_snapshot(callback)
  → L1 大额 Swap → MarketSnapshot
  → callback 内 run_pso_on_snapshot
  → 写 data/results/live_log.jsonl  【待实现】
```

### 多 GPU 模式【待实现】

```
torchrun --nproc_per_node=K scripts/run_ddp.py
  → init_distributed()
  → rank 0 加载 snapshot，广播
  → 每 rank create_pso_optimizer(particles // K)
  → search() 内含 all_reduce gbest
  → rank 0 保存结果
```

## 7. 错误处理

| 场景 | 处理 |
|------|------|
| WebSocket 断线 | 重连；ETH 失败回退 HTTP |
| RPC 401/403 | 检查 Infura Secret |
| CUDA OOM | 减少粒子数 |
| 多 GPU 不同步 | 检查 `torchrun` 与 `dist.init_process_group` |
| 适应度全负 | 输出「无套利机会」，非错误 |
