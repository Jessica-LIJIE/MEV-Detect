# 项目说明

## 1. 课题定位

本项目基于 NDSS 2025 论文 **《MEVisor: High-Throughput MEV Discovery in DEXs with GPU Parallelism》**，构建一套 **L1-L2 跨层 MEV 检测与可视化系统**。


| 维度  | MEVisor（基线）         | 本项目（改进）                                   |
| --- | ------------------- | ----------------------------------------- |
| 场景  | 单链、单 DEX 高性能 MEV 搜索 | **L1（Ethereum）↔ L2（Arbitrum）跨层原子套利** |
| 算法  | 遗传算法（GA）+ GPU       | **向量化粒子群（PSO）**，消除 GA 串行依赖                |
| 并行  | 单链 GPU 矩阵运算         | **真多 GPU**（`torch.distributed` 粒子分片）      |
| 输出  | 套利路径搜索结果            | 检测结果 + **可视化面板**（价差、触发、策略、性能）             |


---

## 2. 老师要求（实验边界）

1. **必须实现真多 GPU**，不能用单卡或 CPU 进程模拟代替性能实验。
2. **换掉遗传算法**——GA 交叉变异存在串行依赖，多 GPU 扩展性差。
3. **用于 L1 与 L2 之间的套利检测**——捕捉跨层价差窗口。
4. **最好能根据 L1/L2 特性适配并行算法**——本项目采用「PSO + 场景化参数配置」实现。
5. 系统形态为 **检测 + 可视化**，不仅是命令行日志。

---

## 3. 改进思路

### 3.1 从单链并行到跨层关联

MEVisor 解决的是「单链内大量路径如何快速搜索」。本项目将搜索对象扩展为：

- L1 价格突变（大额 Swap、区块延迟）作为 **触发信号**
- L2 上同步搜索最优 `Amount_in`、交易路由、跨链桥路径
- 用 `l2_lag_ms` 刻画 L1→L2 时序差，评估套利窗口

### 3.2 算法选择：向量化 PSO


| 特性       | GA          | PSO（本项目）           |
| -------- | ----------- | ------------------ |
| 个体/粒子更新  | 交叉、变异，强顺序依赖 | 速度/位置矩阵更新，无粒子级 for |
| GPU 友好度  | 低           | 高（fitness 批量计算）    |
| 多 GPU 扩展 | 难           | 粒子分片 + 全局 gbest 同步 |


### 3.3 L1/L2 自适应 PSO 配置（E3c，已实现）


| 场景  | L1 特性           | L2 特性    | PSO 策略     |
| --- | --------------- | -------- | ---------- |
| 大价差 | Bundle 竞价激烈，需广搜 | 利润高      | 粒子数 ↑、迭代 ↑ |
| 低延迟 | 窗口短             | 块快、Gas 低 | 迭代 ↓、早停更激进 |


通过 `select_pso_profile(snapshot)` 根据 `spread`、`l2_lag_ms` 动态选择参数，在报告中说明「同一可并行算法、不同链特性不同配置」。

### 3.4 多 GPU 架构

```
总粒子 N = 1000，4 张 GPU：
  GPU0: 粒子 0..249    GPU1: 粒子 250..499
  GPU2: 粒子 500..749  GPU3: 粒子 750..999

每轮迭代：
  1. 各卡本地计算 fitness、更新 pbest
  2. all_reduce 取全局最优 fitness
  3. broadcast 同步 gbest_position
  4. 各卡继续速度/位置更新
```

多 GPU 性能实验在 **Linux 云主机（2～4 卡）** 上完成，本地负责开发、Mock、监听与可视化。

## 6. 简化假设（报告中需说明）

本项目为 **检测框架**，非链上执行系统：

- 利润计算基于价差 × 投入量的简化模型，非完整 Uniswap V3 tick 级仿真
- 桥费使用配置默认值 × 路由系数，非实时桥协议报价
- 「原子套利」指检测层面的跨层可盈利窗口，不包含实际 Bundle 提交

这些简化不影响「算法并行性对比」和「跨层检测流程」的实验目标。

## 7. 老师反馈版（课题重定位）

本节整理老师反馈后的**问题定义、方法框架、数据与实验纪律**。本文档主角是**被前作绕过的跨层非原子结构**；PSO 与 GPU 是匹配该结构的工具，而非独立算法卖点。

### 7.1 背景与动机

去中心化交易所（DEX）上的套利长期是链上价值（MEV）的重要来源。已有工作（如 MEVisor）多聚焦**单链、单一交易对**的价差捕获——在该最简单场景下，「投入多少资金最优」往往存在闭式解，本不需要复杂搜索。

真实套利收益常隐藏在**跨越多个流动性池、多种代币的环形兑换路径**中。随着 Ethereum（L1）与 Arbitrum 等 L2 并行运行，同一资产在不同层、不同协议、不同费率档之间持续存在价格偏离，形成更复杂的跨层套利空间。

**多池套利**与**跨层价差**本身并不新颖；本课题的难点来自一个被以往工作常绕过的事实：

| 结构 | 含义 | 建模后果 |
|------|------|----------|
| **非原子性** | L1↔L2 套利无法在一笔交易内原子完成 | 不能假设闪电贷「资本无穷」 |
| **分层库存约束** | 套利者须预先在 L1、L2 各自部署资金 | 资本成为受限状态变量 |
| **桥接时延风险** | 跨层转移需经历桥接窗口（分钟～数日） | 收益为需权衡风险的期望值，而非价格的确定函数 |

我们将上述两点统一建模为**带分层库存预算与时延风险项**的优化目标，把「跨层非原子」从定性观察落实为可计算的约束与代价。

### 7.2 问题形式

由此形成**混合整数、非凸、非光滑**优化问题：

- **离散层：** 在指数级环路与多个平行池间做组合选择；
- **连续层：** Uniswap V3 集中流动性导致分段、梯度不连续，传统凸优化与梯度法不适用。

该结构正是元启发式搜索的用武之地——也正是在这里，单链闭式场景下显得多余的「快速搜索」才第一次变得**真正必要**。

### 7.3 两阶段求解框架

```
[L1/L2 多池真实快照]
        │
        ▼
┌───────────────────────────────┐
│ 阶段1：候选环筛选              │
│ 跨层 token-池图 + 负对数汇率   │
│ 有效深度过滤 → Top-K 候选环    │
│ （可采纳松弛：不漏盈利、少误杀）│
└───────────────┬───────────────┘
                ▼
┌───────────────────────────────┐
│ 阶段2：向量化 PSO 资金分配     │
│ 在 K 条候选环上搜 amount 等    │
│ 无梯度、逐元素更新、易张量化   │
└───────────────┬───────────────┘
                ▼
        [套利策略 + fork 验真]
```

**PSO 的定位（需在报告中坦诚）：** 与问题结构相匹配的实现工具，而非理论更优算法；相对 GA 的优势主要来自**向量化实现**，E1 需用闭式/三元搜索上界验证解质量。

**多 GPU 的定位：** 仅当阶段2 的批量 fitness 在**单卡上已吃满**时再测多卡；若规模不足，**如实报告「多卡不适用」** 的负结果（与旧版单池 E2 结论一致且合理）。

### 7.4 数据与验真（回应「多池数据不好找」）

不依赖 MEVisor 等未公开的 benchmark；**实验数据全部来自链上真实状态的自建快照**，辅以少量可控 Mock，**二者分开使用、分开报告**。

| 类型 | 用途 | 要点 |
|------|------|------|
| **真实快照** | E2/E3 主实验 | Multicall3 批量拉 slot0、liquidity、tickBitmap/ticks；eth_call **钉死同一区块号**；跨层用 **L2 block→L1 batch 映射**（非简单时间戳） |
| **Mock 案例** | E1 消融、PSO 收敛验证 | 合成价差/深度/桥时延已知场景；**Mock 利润不作为实验发现报告** |
| **fork 回放** | 利润验真 | anvil `--fork` 同区块执行抽样路由；QuoterV2 离线校验模拟一致性 |

池清单聚焦高流动性交易对（WETH/USDC/USDT/DAI/ARB 等），以控制图规模与 tick 重建成本。桥接费率与时延参数取自真实桥合约配置（慢提款桥、Across/Hop 等）。

**三类证据的分工：**

- 真实快照 → 「机会是否存在」
- fork 回放 → 「利润是否可信」
- Mock → 「算法是否正确」

### 7.5 实验设计（老师版 E1～E4）

与初版实验清单的编号相同，**含义已重定义**：

| 编号 | 名称 | 要回答的问题 | 核心指标 |
|------|------|--------------|----------|
| **E1** | 算法对照 | 求解器是否值得 | 墙钟时间；净利润 / 闭式上界 |
| **E2** | GPU 可扩展性 | 并行到何处为止 | 单卡「规模 vs 墙钟」曲线；必要时 island 多卡 vs 每代 sync |
| **E3** | 检测与验真 | 机会是否真实、利润是否可信 | 检出环数、利润分布、延迟；fork 实测 vs 模拟误差 |
| **E4** | 威胁与限定 | 结论适用范围 | 检测≠捕获；无真值标签故不报 PR/Recall；λ 与库存成本敏感性 |

**贯穿纪律：** 真实快照与 Mock 分开报告；计时经预热后报均值±标准差；涉及求解的实验报告**解质量分布**，非仅均值利润。
