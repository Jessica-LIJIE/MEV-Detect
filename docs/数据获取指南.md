# 数据获取指南

说明 Mock、实时链上与历史数据的获取方式，配合 [实验清单.md](./实验清单.md) E3 使用。

---

## 1. 数据需求总览

| 数据类型 | 用途 | 获取方式 | 对应实验 |
|----------|------|----------|----------|
| Mock 快照 | 离线开发、E1/E2/E3a | `data/mock_mempool.json` | E1、E2、E3a |
| 实时区块/Swap | Live 监听、E3b | WebSocket / HTTP RPC | E3b |
| 池子链上状态 | 价格、流动性 | `eth_call` slot0 | E3 |
| Gas 价格 | 成本模型 | `eth_gasPrice` | E3 |
| Benchmark 原始数据 | 多 GPU 复现 | `scripts/benchmark_multi_gpu.py` 输出 | E2 |
| Live 日志 | 可视化面板 | `data/results/live_log.jsonl` | E4 |
| 历史回测（可选） | 补充案例 | Dune / Arbiscan | E3 加分 |

## 4. Mock 数据

### 4.1 文件位置

`data/mock_mempool.json` — 10 条跨层场景，含正/负利润案例。

### 4.2 单条记录结构

```json
{
  "snapshot_id": "snap_001",
  "scenario": "L1 大涨，L2 滞后",
  "timestamp": "2025-05-20T10:00:00Z",
  "trigger": "l1_large_swap",
  "l1": {
    "chain": "ethereum",
    "block_number": 18500000,
    "eth_usdc_price": 3520.0,
    "sqrt_price_x96": "...",
    "liquidity": "...",
    "tick": 0
  },
  "l2": {
    "chain": "arbitrum",
    "block_number": 150000000,
    "eth_usdc_price": 3505.0
  },
  "l2_lag_ms": 1500,
  "gas": { "l1_gwei": 25.0, "l2_gwei": 0.1 },
  "bridge_fee_usd": 5.0,
  "swap_amount_usd": 80000
}
```

### 4.3 校验

```bash
python scripts/validate_mock_data.py
python main.py --mock --record-id snap_005
```

### 4.4 设计原则

- 至少 3 条应有正价差且扣费后盈利  
- `l2_lag_ms` 覆盖 200～3000 ms，供自适应 PSO 测试  
- `scenario` 字段便于报告引用

## 5. 实时数据（E3b）

### 5.1 监听命令

```bash
python main.py --live --http --duration 3600
```

### 5.2 触发条件

- L1 Swap 折算 USD ≥ `LISTENER_CONFIG["l1_swap_threshold_usd"]`（默认 50000）  
- 演示时可临时调低阈值（仅本地，勿提交敏感配置）  

### 5.3 国内网络建议

- Ethereum WebSocket 易超时 → 使用 `--http`  
- Arbitrum WebSocket 通常可用  

### 5.4 输出【待实现】

每次触发写入 `data/results/live_log.jsonl`，供 Streamlit 读取。

## 6. 历史数据（可选）

### 6.1 Arbiscan / Etherscan

- 按池子地址导出 Swap 事件  
- 用于补充 1～2 个真实时间戳案例  

### 6.2 Dune Analytics

- 查询 Uniswap V3 ETH/USDC 跨链价差（SQL 社区查询）  
- 适合报告「真实市场曾出现价差窗口」的旁证  

### 6.3 回测流程

1. 将历史记录转为 `MarketSnapshot` 格式  
2. `python main.py --mock` 或单条 `--record-id`  
3. 记录 PSO 输出与耗时
